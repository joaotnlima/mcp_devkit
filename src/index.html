Test Runner

Test your MCP servers like you test your code

Write declarative tests in .mcptest files. Run them locally or in CI. Get instant feedback with rich terminal output. Like Jest, but for MCP server testing.

weather.mcptest 
server "weather-server" {
  transport: "stdio"
  command:  "node server.js"
}

test "get_weather returns forecast" {
  call tools/get_weather { city: "London" }
  expect status 200
  expect result.temperature is_number
  expect result.conditions is_string
}

test "handles missing params" {
  call tools/get_weather {}
  expect error.code -32602
}

Frequently asked questions

What is MCP DevKit?
MCP DevKit is an open-source developer toolkit for testing, debugging, inspecting, and monitoring MCP (Model Context Protocol) servers. Think of it as Jest + Postman + Datadog for the Model Context Protocol. It includes a CLI test runner, request inspector, schema validator, and production monitoring.
How do I test an MCP server?
Run npx mcp-devkit init to scaffold test files, write declarative tests in .mcptest files, and run them with npx mcp-devkit test. Tests can verify tool calls, resource reads, and prompt execution. Works locally and in CI/CD pipelines.
Does MCP DevKit work with Claude, GPT, and other LLMs?
Yes. MCP DevKit tests your MCP server independently of any specific LLM. Since MCP is a protocol standard, any server that speaks MCP works with MCP DevKit — whether your agents run on Claude, GPT, LangChain, CrewAI, or custom frameworks.
Is MCP DevKit free?
The CLI tools are open-source and free forever under the MIT license. Local testing, inspection, and validation cost nothing. Cloud features (production monitoring, registry, team dashboards) will have a free tier with paid plans for teams.
Can I use MCP DevKit in CI/CD?
Absolutely. MCP DevKit is designed for CI/CD from day one. Run npx mcp-devkit test in your GitHub Actions, GitLab CI, or any pipeline. It exits with proper codes, outputs structured results, and supports JUnit XML reporting.
How is MCP DevKit different from MCP Inspector?
MCP Inspector is great for quick manual debugging of a single server. MCP DevKit is a full testing and monitoring platform: automated test suites, CI/CD integration, schema validation, production monitoring, and a server registry. Inspector is a screwdriver; DevKit is the whole toolbox.
What is the .mcptest file format?
The .mcptest format is a declarative test specification for MCP servers. You define test cases that describe expected tool behavior, resource schemas, and prompt outputs. It reads like English and compiles to protocol-level assertions.
Does MCP DevKit work with stdio, SSE, and HTTP transports?
Yes. MCP DevKit supports all three MCP transport types: stdio (local process), SSE (Server-Sent Events), and HTTP (streamable). Configure the transport in your test file or CLI flags.
